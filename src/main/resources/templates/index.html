Mais, la qualité de code, qu'est-ce que c'est?

La qualité de code est un ensemble de critères qui permettent de juger de la qualité d'un code source.

Il existe différentes méthodes permettant de rendre un code "propre"

Pour simplifier, il faut respecter les principes suivants :

1. Ne pas écrire de "franglais", les thermes doivent tous être en anglais

2. Ne pas utiliser de magic numbers.
Un "magic number" est une valeur numérique qui apparaît dans le code source sans explication ou justification claire de sa signification.

3. Les nommages doivent respecter une certaine convention.
Il faut entre autre que le nom d'une classe doit décrire sa fonction, et que les noms des variables doivent être explicites.
